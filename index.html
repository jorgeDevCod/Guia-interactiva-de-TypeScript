Inicio / Nivel 1 / Tipos básicos

🔵 Tipos básicos

El núcleo de TypeScript. Si no entendés estos tipos, vas a sufrir en cualquier framework.

Los tipos primitivos

TypeScript agrega tipado estático a JavaScript. Los tipos básicos son:

// Tipos primitivos
let nombre: string = "María";
let edad: number = 28;
let activo: boolean = true;
let nada: null = null;
let sinDef: undefined = undefined;

// Arrays
let nums: number[] = [1, 2, 3];
let nombres: Array<string> = ["Ana", "Luis"];

// Tuplas
let par: [string, number] = ["edad", 30];

⚠️ any vs unknown — La diferencia crítica

🚫
any desactiva completamente TypeScript. Si usás any, le estás diciendo a TS "no me controles esto". Es como no usar TypeScript.
✅
unknown es la alternativa segura. TS te obliga a validar el tipo antes de usar el valor. Es el tipo correcto para valores que no conocés de antemano (respuestas de API, valores externos).
// ❌ any — evitar siempre
let dato: any = fetchData();
dato.toUpperCase(); // TS no protesta, pero puede romper en runtime

// ✅ unknown — el correcto
let dato2: unknown = fetchData();
// dato2.toUpperCase(); // ❌ Error: necesitás validar primero

if (typeof dato2 === "string") {
  dato2.toUpperCase(); // ✅ Ahora TS sabe que es string
}

void y never

// void — funciones que no retornan nada
function log(msg: string): void {
  console.log(msg);
}

// never — funciones que nunca terminan o siempre lanzan error
function lanzarError(msg: string): never {
  throw new Error(msg);
}

// never también en exhaustive checks
function assertNever(x: never): never {
  throw new Error("Caso no manejado: " + x);
}

Type Assertions

// Cuando vos sabés más que TS
const input = document.getElementById("email") as HTMLInputElement;
const valor = input.value; // TS sabe que es HTMLInputElement

// Alternativa con genéricos (no funciona en JSX)
const input2 = <HTMLInputElement>document.getElementById("email");
💡
TypeScript infiere tipos automáticamente. Si escribís let x = 5, TS sabe que x es number. Solo necesitás anotar tipos cuando TS no puede inferirlos o cuando querés ser explícito.
Inicio / Nivel 1 / Funciones

⚙️ Funciones tipadas

Tipar funciones correctamente es fundamental para cualquier codebase TypeScript.

Tipado básico de funciones

// Función declarada
function suma(a: number, b: number): number {
  return a + b;
}

// Arrow function
const resta = (a: number, b: number): number => a - b;

// Parámetros opcionales y con default
function saludar(nombre: string, saludo?: string): string {
  return `${saludo ?? "Hola"}, ${nombre}!`;
}

// Resto de parámetros
function sumarTodos(...nums: number[]): number {
  return nums.reduce((a, b) => a + b, 0);
}

Funciones como tipos

// Definir un tipo para funciones
type Operacion = (a: number, b: number) => number;

const multiplicar: Operacion = (a, b) => a * b;

// Función como parámetro (callback)
function aplicar(a: number, b: number, operacion: Operacion): number {
  return operacion(a, b);
}

// Función que retorna función
function multiplicarPor(factor: number): (n: number) => number {
  return (n) => n * factor;
}
const porDos = multiplicarPor(2);
porDos(5); // 10

Overloads

// Misma función, distintos tipos de parámetros
function formatear(valor: number): string;
function formatear(valor: string): string;
function formatear(valor: number | string): string {
  if (typeof valor === "number") return valor.toFixed(2);
  return valor.trim();
}

ReturnType — Inferir el tipo de retorno

function getUser() {
  return { id: 1, name: "Ana", role: "admin" as const };
}

// ReturnType infiere el tipo de retorno de getUser
type User = ReturnType<typeof getUser>;
// { id: number; name: string; role: "admin" }

📦 Objetos e Interfaces

Cómo modelar estructuras de datos complejas con TypeScript.

Interface básica

interface User {
  id: number;
  name: string;
  email: string;
  age?: number; // Opcional
  readonly createdAt: Date; // No se puede modificar
}

const user: User = {
  id: 1,
  name: "Carlos",
  email: "[email protected]",
  createdAt: new Date()
};

// user.createdAt = new Date(); // ❌ Error: readonly

Extender interfaces

interface Animal {
  nombre: string;
  edad: number;
}

interface Perro extends Animal {
  raza: string;
  ladrar(): void;
}

// Extender múltiples interfaces
interface AdminUser extends User, Timestamps {
  role: "admin";
  permissions: string[];
}

Intersection Types (&)

type Timestamps = {
  createdAt: Date;
  updatedAt: Date;
};

type UserWithTimestamps = User & Timestamps;
// Tiene TODOS los campos de User MÁS los de Timestamps

Index Signatures

interface Dictionary {
  [key: string]: string;
}

const colores: Dictionary = {
  rojo: "#ff0000",
  verde: "#00ff00",
};

// Objetos como mapas con Record (más limpio)
type ColoresMap = Record<string, string>;

Interface con métodos

interface Repository<T> {
  findById(id: number): Promise<T | null>;
  findAll(): Promise<T[]>;
  save(entity: T): Promise<T>;
  delete(id: number): Promise<void>;
}
Inicio / Nivel 1 / Type vs Interface

⚖️ Type vs Interface

La pregunta más frecuente en entrevistas de TypeScript. Acá la respuesta definitiva.

Las diferencias clave

Característica interface type
Objetos ✅ Ideal ✅ Funciona
Primitivos ❌ No puede ✅ type ID = string | number
Uniones ❌ No puede ✅ type A = B | C
Extender ✅ extends ✅ & (intersection)
Declaration Merging ✅ Sí (reopening) ❌ No
Implementar en clase ✅ implements ✅ implements (si es objeto)
Mapped Types ❌ No puede ✅ Sí

Cuándo usar cada uno

📌
Regla práctica de la industria: interface para objetos y clases, type para todo lo demás (unions, primitivos, mapped types, utility types).
// ✅ interface — ideal para objetos/contratos
interface User {
  id: number;
  name: string;
}

// ✅ type — ideal para uniones y aliases
type Status = "pending" | "active" | "inactive";
type ID = string | number;
type Callback = () => void;

// Declaration Merging (solo interfaces)
interface Window {
  myCustomProp: string; // Agrega a Window global
}

// type no puede hacer esto:
// type Window = { ... } // ❌ Duplicate identifier

Lo que type puede hacer y interface no

// Tipos condicionales
type IsString<T> = T extends string ? true : false;

// Mapped types
type Optional<T> = {
  [K in keyof T]?: T[K];
};

// Template literal types
type EventName = `on${Capitalize<string>}`;
Inicio / Nivel 1 / Union Types

🔀 Union Types y Narrowing

Tipos que pueden ser uno de varios valores, y cómo discriminarlos en runtime.

Union Types básicos

let id: string | number;
id = "abc123"; // ✅
id = 42;       // ✅

type Status = "loading" | "success" | "error";
type Color = "rojo" | "verde" | "azul";

Narrowing con typeof

function procesar(valor: string | number): string {
  if (typeof valor === "string") {
    return valor.toUpperCase(); // TS sabe: string
  }
  return valor.toFixed(2); // TS sabe: number
}

Discriminated Unions (patrón esencial)

type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "square"; side: number }
  | { kind: "triangle"; base: number; height: number };

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "square":
      return shape.side ** 2;
    case "triangle":
      return (shape.base * shape.height) / 2;
  }
}

Type Guards personalizados

interface Dog { bark(): void; }
interface Cat { meow(): void; }

function isDog(animal: Dog | Cat): animal is Dog {
  return "bark" in animal;
}

function makeSound(animal: Dog | Cat) {
  if (isDog(animal)) {
    animal.bark(); // TS sabe que es Dog
  } else {
    animal.meow(); // TS sabe que es Cat
  }
}
Inicio / Nivel 1 / Generics

🧬 Generics

El feature más poderoso de TypeScript. Imprescindible para React y APIs robustas.

¿Qué son los Generics?

Permiten escribir código que funciona con cualquier tipo, manteniéndose type-safe. Es como un "placeholder" de tipo.

// Sin generics — repetido, no type-safe
function identityStr(val: string): string { return val; }
function identityNum(val: number): number { return val; }

// Con generics — una sola función, completamente type-safe
function identity<T>(val: T): T { return val; }

identity<string>("hola"); // T = string
identity(42);            // TS infiere T = number

Generics en funciones con constraints

// T debe tener la propiedad length
function getLength<T extends { length: number }>(item: T): number {
  return item.length;
}

getLength("hola");   // ✅ 4
getLength([1,2,3]); // ✅ 3
// getLength(42);   // ❌ number no tiene length

// Múltiples type params
function pair<A, B>(a: A, b: B): [A, B] {
  return [a, b];
}

// keyof constraint
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

Generics en interfaces y tipos

// Interface genérica
interface ApiResponse<T> {
  data: T;
  status: number;
  message: string;
}

// Usar con tipos concretos
type UserResponse = ApiResponse<User>;
type ListResponse<T> = ApiResponse<T[]>;

// Promesas tipadas
async function fetchUser(id: number): Promise<User> {
  const res = await fetch(`/api/users/${id}`);
  return res.json();
}
⚛️
En React, los generics son esenciales para hooks: useState<User | null>, useRef<HTMLInputElement>. Los verás constantemente.
Inicio / Nivel 1 / Utility Types

🛠️ Utility Types

Transformadores de tipos que usarás todos los días en proyectos reales.

interface User {
  id: number;
  name: string;
  email: string;
  password: string;
  age: number;
}

Partial<T> — Todos opcionales

type UpdateUser = Partial<User>;
// { id?: number; name?: string; email?: string; ... }

// Caso de uso real: función de update
function updateUser(id: number, data: Partial<User>) {
  // Solo actualiza los campos provistos
}

Pick<T, K> y Omit<T, K>

// Pick — elegir campos
type UserPreview = Pick<User, "id" | "name">;
// { id: number; name: string }

// Omit — excluir campos
type PublicUser = Omit<User, "password">;
// { id: number; name: string; email: string; age: number }

// CreateUser — sin id (se genera en backend)
type CreateUser = Omit<User, "id">;

Record<K, V>

// Mapa de tipos
type Traducciones = Record<string, string>;
type StatusMessages = Record<"loading" | "error" | "success", string>;

const msgs: StatusMessages = {
  loading: "Cargando...",
  error: "Algo salió mal",
  success: "¡Listo!"
};

Required, Readonly, NonNullable

Required<Partial<User>>  // todos requeridos
Readonly<User>          // no se puede mutar
NonNullable<string | null | undefined> // = string

// Extraer tipo de Promise
type PromiseValue<T> = T extends Promise<infer U> ? U : T;
type Awaited<T> = T extends Promise<infer R> ? R : T;

⚛️ TypeScript + React — Props, Hooks y Eventos

Donde la mayoría falla en entrevistas. Todo lo que necesitás para React profesional.

Tipar Props de componentes

interface ButtonProps {
  label: string;
  onClick: () => void;
  disabled?: boolean;
  variant?: "primary" | "secondary" | "danger";
  children?: React.ReactNode;
}

const Button = ({ label, onClick, disabled = false, variant = "primary" }: ButtonProps) => (
  <button onClick={onClick} disabled={disabled} className={variant}>
    {label}
  </button>
);

// React.FC — alternativa (menos popular hoy)
const Button2: React.FC<ButtonProps> = ({ label }) => <button>{label}</button>;

useState tipado

const [count, setCount] = useState<number>(0);
const [user, setUser] = useState<User | null>(null);
const [items, setItems] = useState<string[]>([]);

// Con objetos
interface FormState {
  email: string;
  password: string;
  isLoading: boolean;
}

const [form, setForm] = useState<FormState>({
  email: "",
  password: "",
  isLoading: false
});

useRef tipado

const inputRef = useRef<HTMLInputElement>(null);
const divRef = useRef<HTMLDivElement>(null);

// Para valores mutables (no DOM)
const counterRef = useRef<number>(0);

// Usar con null check
const focusInput = () => {
  inputRef.current?.focus();
};

Eventos tipados

// Input change
const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  setForm(prev => ({ ...prev, email: e.target.value }));
};

// Select change
const handleSelect = (e: React.ChangeEvent<HTMLSelectElement>) => {};

// Button click
const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => {};

// Form submit
const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
  e.preventDefault();
};

Custom Hooks tipados

function useLocalStorage<T>(key: string, initial: T): [T, (val: T) => void] {
  const [state, setState] = useState<T>(() => {
    const stored = localStorage.getItem(key);
    return stored ? JSON.parse(stored) : initial;
  });

  const setStorage = (val: T) => {
    localStorage.setItem(key, JSON.stringify(val));
    setState(val);
  };

  return [state, setStorage];
}

// Uso
const [theme, setTheme] = useLocalStorage<"light" | "dark">("theme", "light");
Inicio / Nivel 2 — React / Hooks

🪝 Hooks tipados — useReducer, useContext

Hooks avanzados con TypeScript para state management real.

useReducer tipado

type Action =
  | { type: "INCREMENT" }
  | { type: "DECREMENT" }
  | { type: "RESET"; payload: number };

interface State { count: number; }

function reducer(state: State, action: Action): State {
  switch (action.type) {
    case "INCREMENT": return { count: state.count + 1 };
    case "DECREMENT": return { count: state.count - 1 };
    case "RESET":     return { count: action.payload };
  }
}

const [state, dispatch] = useReducer(reducer, { count: 0 });

useContext tipado

interface AuthContext {
  user: User | null;
  login: (credentials: Credentials) => Promise<void>;
  logout: () => void;
}

const AuthCtx = createContext<AuthContext | null>(null);

// Custom hook para acceder al context con type guard
function useAuth(): AuthContext {
  const ctx = useContext(AuthCtx);
  if (!ctx) throw new Error("useAuth fuera de AuthProvider");
  return ctx;
}
Inicio / Nivel 2 — React / Eventos

🎯 Eventos, forwardRef y más patrones React

forwardRef tipado

interface InputProps {
  label: string;
  error?: string;
}

const Input = React.forwardRef<HTMLInputElement, InputProps>(
  ({ label, error }, ref) => (
    <div>
      <label>{label}</label>
      <input ref={ref} />
      {error && <span>{error}</span>}
    </div>
  )
);

ComponentProps — reutilizar tipos nativos

// Extender props nativas de un elemento HTML
type ButtonProps = React.ComponentProps<"button"> & {
  variant?: "primary" | "secondary";
  isLoading?: boolean;
};

// Spread operator para props nativas
const Button = ({ variant, isLoading, ...rest }: ButtonProps) => (
  <button {...rest} disabled={isLoading || rest.disabled} />
);
